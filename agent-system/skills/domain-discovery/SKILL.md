# Domain Discovery

## Objective

Create shared understanding before execution starts.

## Steps

1. Define domain boundaries and terminology.
2. List key actors, workflows, constraints.
3. Identify unknowns and assumptions.
4. Convert findings into actionable requirements.

## Output

- Domain glossary
- Core workflows
- Assumptions and unknowns
- Discovery-to-plan handoff notes

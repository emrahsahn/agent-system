Project Overview

Agent Workflow Starter Kit

A platform-agnostic repository for building with specialized AI agents and reusable skills. The system separates planning, implementation, and review to keep context clean and output quality high.

What This Project Includes

agents/

Specialist roles such as planner, ui-agent, builder, and reviewer.

skills/

Reusable behavior packs for testing, code review, and UI/UX decisions.

cursor-agent-system/

Expanded role system with orchestration, engineering, product, security, and research agents.

Core Workflow

1) Plan

Define scope, architecture, risks, and execution steps.

2) Design

Specify UI structure and decision rationale.

3) Build

Implement with a test-first mindset when possible.

4) Review

Run quality and security checks before merge.

Why This Structure Works

  • Single-purpose agents reduce context mixing and mistakes.
  • An orchestrator can coordinate tasks while specialists stay focused.
  • Reusable skills standardize quality across projects.
  • The setup is portable across different AI coding environments.

Quick Start (Generic)

Copy the provided definitions into your own agent runtime directories.

cp agents/*.md <your-agent-directory>/ cp -r skills/* <your-skills-directory>/ cp -r cursor-agent-system/agents/*.md <your-agent-directory>/ cp -r cursor-agent-system/skills/* <your-skills-directory>/

Suggested Public Repo Positioning

If you publish this on GitHub, position it as an open starter framework for multi-agent development workflows rather than a tool-specific template.

[ 01 / WHAT IS AN AGENT? ]

> WHAT IS AN AGENT, AND WHY USE IT?

Agent = a Claude instance focused on one job. It has its own system prompt, allowed toolset, and expertise.

Why not only one Claude?
If one conversation handles planning + design + implementation + security at once, context gets noisy. Decisions from earlier files are lost, and quality drops.

Better approach: split the work across 4 specialists. Each one keeps a narrow context and a focused task.

[ 02 / 4 AGENTS ]

> PLAN → DESIGN → BUILD → REVIEW

This is a natural web delivery flow. One agent owns each stage.

[ AGENT · 01 · PLAN ]
planner
Plans the work. Breaks features into steps, identifies risks, and defines architecture.
[ AGENT · 02 · DESIGN ]
ui-agent
Defines visual direction and component structure. Layout, color, and typography decisions.
[ AGENT · 03 · BUILD ]
builder
Implements most of the code. Translates planning + UI decisions into working output.
[ AGENT · 04 · REVIEW ]
reviewer
Validates output quality and security. Final gate before commit.

Key idea: each agent has one responsibility. Context stays clean and responsibilities do not overlap.

[ 03 / WHAT IS A SKILL? ]

> SKILL = BEHAVIOR PACKAGE FOR AN AGENT

A skill = folder + SKILL.md file. It defines when to trigger, how to behave, and which output format to follow.

When your assistant tooling sees a matching scenario, it loads the skill and adds those instructions to context.

/path/to/skills/ ├── tdd/ │ └── SKILL.md single-file basic skill ├── ui-ux-pro-max/ richer skill package │ ├── SKILL.md main instruction file │ ├── styles.md reference · 50+ styles │ ├── palettes.md reference · 161 color palettes │ └── components.md reference · component examples └── security-review/ └── SKILL.md

Example SKILL.md structure (YAML frontmatter + markdown instructions):

---
name: ui-ux-pro-max
description: Use when designing UI/UX. 50+ styles, 161 palettes, components.
---
# UI/UX Pro Max
## Trigger Conditions
- When user requests UI/UX design
- When "component", "layout", or "design system" appears
## Approach
1. Pick a style from styles.md
2. Pick colors from palettes.md
3. Reuse patterns from components.md

[ 05 / SKILL SOURCES ]

> WHERE DO YOU GET SKILLS?

You do not need to write everything from scratch. The GitHub community shares 1000+ skills. Three options:

  • 1 · Clone an existing repository
    git clone https://github.com/anthropics/skills ~/anthropic-skills
    then copy the skill folders you need into your skills directory
  • 2 · Select a single skill (if you do not need full packs)
    Search via VoltAgent/awesome-agent-skills and copy only what you need.
  • 3 · Write your own (about 5 minutes)
    mkdir your-skills/my-skill && echo "..." > SKILL.md
    YAML frontmatter + markdown = ready skill.

Suggested baseline: anthropics/skills (official) + curated community picks + custom skills for your workflow.

[ 06 / COMMUNICATION HIERARCHY ]

> HOW DO AGENTS COMMUNICATE?

Agents do not talk directly to each other. There is a strict hierarchy - everything flows through Main Claude.

[ ORCHESTRATOR ]
MAIN CLAUDE
↓ Task ↓ Task ↓ Task ↓ Task
▼
planner
returns one summary message
▼
ui-agent
returns one summary message
▼
builder
returns one summary message
▼
reviewer
returns one summary message
↑ SHARED FILES READ BY ALL AGENTS ↑
CLAUDE.md
global rules
stack preferences
agents/*.md
agent definitions
roles + tools
rules/*.md
coding style
test standards
skills/*/
behavior packages
auto-loaded

Important: sub-agent intermediate work (tool calls, temporary reads) is discarded. Only one summary message returns to Main Claude.

[ 07 / REAL FLOW ]

> STEP-BY-STEP CALL

How does Main Claude call a sub-agent with the Task tool? Terminal perspective:

[00:00][main] user: "Review the pricing page"
[00:01][main] calls specialist: reviewer("review src/pricing/* code")
[00:02][reviewer] new context started
[00:02][reviewer] auto-loaded: CLAUDE.md, rules/*, skills/code-review, skills/security-review
[00:03][reviewer] Read: src/pricing/page.tsx
[00:04][reviewer] Read: src/pricing/api.ts
[00:05][reviewer] Grep: "webhook" across files
[00:07][reviewer] found: 2 issues · 1 CRITICAL (missing webhook signature)
[00:08][reviewer] RETURNS SUMMARY -> main
[00:08][system] reviewer context deleted (24 tool calls · 2400 tokens)
[00:09][main] received: "1 CRITICAL: missing webhook signature · line 47"
[00:09][main] sends summary to user

The main chat sees only one summary line. No 24-tool-call noise, and context stays clean.

[ 08 / LIVE EXAMPLE ]

> 4 AGENTS · WORKING TOGETHER

Scenario: "Build a pricing page - Stripe, 3 plans". Main Claude calls 4 agents in sequence.

[user] "Build pricing page · Stripe + 3 plans"
[01][main] orchestrator -> planner: create execution steps
[02][planner] returns 6-step plan · risks: webhook security
[03][main] orchestrator -> ui-agent: component and layout decisions
[04][ui-agent] returns 3 plan cards + monthly/yearly toggle
[05][main] orchestrator -> builder: tests + implementation
[06][builder] 12 tests written · code added · tests passing
[07][main] orchestrator -> reviewer: quality + security review
[08][reviewer] 1 CRITICAL: webhook signature not verified · line 47
[09][main] orchestrator -> builder: "fix webhook signature"
[10][builder] fix applied · tests passing
[11][main] orchestrator -> reviewer: re-check
[12][reviewer] CLEAN · ready to commit

Notice: Main Claude is the orchestrator. Each agent does one job and returns one summary.
